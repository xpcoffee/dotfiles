# Report style rules

The report is evidence the user reads about their own week. It must survive a bad week without becoming either a flogging or a comfort blanket: state what happened, link the artefact, stop.

Non-negotiable:

- **No verdicts, no advice, no synthesis.** No Summary, no Highlights, no Risks, no "consider doing X", no "overall". Moments, coverage, gaps – the user writes the conclusions.
- **No unsupported causal claims.** Scan for "used", "applied", "based on", "drove", "led to", "because of", "resulted in". Each must point at an artefact showing the link. "Sent the summary" is a fact; "the summary unblocked the team" needs the message where someone says so.
- **Asks vs does.** A stated intention is a commitment; completion is tagged only with the artefact that closed it.
- **No emotive adverbs** ("quickly", "carefully", "proactively", "thoughtfully", "promptly") – state the substance and the timestamp instead.
- **No adverbial disjuncts** ("clearly", "plainly", "simply", "unfortunately", "notably") – state the claim directly.
- **No mind-reading.** Not "avoided the discussion", "lost interest", "deprioritised" – write the observable: "did not reply in the thread; no related artefact found this week".
- **Every quote appears in the source.** No invented or paraphrased-as-verbatim quotes.
- **Every dimension tag points at a specific moment.** A tag without a moment is noise.
- **People as `@firstname-lastname`** matching files in `people/`; check spelling against existing filenames. Moments involving others describe the artefact and interaction, never characterise the other person.
- **No em dashes**; commas, colons, full stops, or en dashes.
- **Links over descriptions**: every moment carries the PR / work item / thread reference so the user can open the source in one click.
